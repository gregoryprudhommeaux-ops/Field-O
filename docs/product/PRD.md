# FIELD-O Product Requirements Document

## Product name
FIELD-O

## Product category
Mobile-first B2B field commissioning and reporting platform

## Vision
FIELD-O helps field teams create professional commissioning and inspection reports quickly, consistently, and with minimal typing, directly from a phone or tablet.

## Core problem
Technicians and field engineers often produce reports too slowly, with too much manual text, inconsistent formatting, and poor traceability. FIELD-O reduces friction and standardizes reporting.

## Primary users
- Maintenance technicians
- Commissioning engineers
- Site supervisors
- Industrial service providers
- Technical project managers

## Secondary users
- Back-office coordinators
- Client stakeholders receiving reports
- QA / operations managers

## Key value proposition
Transform field inspections into structured, branded, shareable reports in minutes.

## Current MVP scope
- Create report
- Choose language EN / ES
- Fill project info
- Complete checklist
- Select predefined common defects for failed items
- Add photo documentation
- Add comments
- Capture signature
- Preview PDF
- Download or share PDF
- Save and review report history
- Support offline draft mode
- Sync local drafts when online
- Select or create client
- Associate reports with clients
- Add company and client logo to PDF
- Handle multi-asset reporting
- Handle technical measurements
- Validate voltage tolerance
- Choose inspection template

## Non-goals for current migration
- Full enterprise auth workflows
- Advanced role management
- Full admin back office
- Billing
- Complex analytics dashboards
- Native iOS / Android apps
- Full AI-assisted narrative generation

## UX constraints
- Must work first on mobile
- Must remain usable with poor connectivity
- Must reduce free typing as much as possible
- Must support review before sending
- Must feel credible for industrial B2B use
- Must support both English and Spanish
- Must be easy to use under time pressure

## Technical constraints
- Single codebase for mobile and desktop
- Offline-capable draft flow
- Firebase-compatible backend
- Modular and maintainable architecture
- Easy future integration with AI assistance and automation

## Success criteria
- A technician can create a clean report quickly
- A manager can review and share the report confidently
- The app works on both desktop and mobile
- The architecture is stable enough to evolve beyond MVP

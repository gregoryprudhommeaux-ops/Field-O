# FIELD-O Design Principles

## Brand personality
- Industrial
- Reliable
- Efficient
- Professional
- International
- Clear under pressure

## Design intent
FIELD-O should feel like a serious field operations tool, not a playful consumer app.

## UI principles
- mobile-first
- large touch targets
- minimum 44px interactive areas
- high contrast
- low friction data entry
- minimal unnecessary text input
- clear next / previous navigation
- obvious report status and sync state
- review before final output
- professional PDF presentation

## Visual direction
- robust B2B visual language
- clean hierarchy
- practical elegance
- restrained use of color
- avoid gimmicky AI aesthetics
- avoid noisy dashboards
- avoid decorative complexity that reduces field usability

## Accessibility
- WCAG AA minimum
- readable under outdoor conditions
- keyboard accessible on desktop
- visible focus states
- strong feedback for errors and sync state

## Device strategy
- primary device: mobile
- secondary device: desktop
- one shared design system
